1.1 操作系统的基本概念

1.1.1 操作系统的概念

定义：操作系统是控制和管理整个计算机系统的硬件与软件资源，合理地组织、调度计算机的工作与资源的分配，进而为用户和其他软件提供方便接口与环境的程序集合，是计算机系统中最基本的系统软件。

实例：常见的操作系统有Windows、macOS、Linux、Android、iOS等。它们都是用来管理计算机资源并为用户和应用程序提供服务的系统软件。

本质：是系统软件（一整套程序构成，如UNIX由上千个模块构成）

核心作用：

  1. 是计算机系统资源的管理者
  2. 为上层提供服务（为用户和其他软件提供更方便的接口和环境）
  3. 是计算机系统中最基础的系统软件（最接近底层硬件的一套软件）

1.1.2 操作系统的特征

操作系统有四个基本特征，其中并发和共享是最基本的特征，二者互为存在条件（没有并发和共享，就没有虚拟和异步）：

记忆方法：

联想记忆法：可以把操作系统四个特征想象成一个高效的团队：

  • 并发（Concurrency） - 团队成员可以同时处理多项任务
  • 共享（Sharing） - 团队成员共享办公设备如打印机、会议室
  • 虚拟（Virtual） - 通过虚拟化技术，让一个物理设备看起来像多个设备
  • 异步（Asynchronism） - 任务完成时间不确定，可能快可能慢

口诀记忆："并共虚异"，可以联想为"并肩共享虚幻异议"

1. 并发：

  • 宏观上同时发生，微观上交替发生
  • 操作系统的并发性：计算机系统"同时"运行着多个程序，宏观上看是同时运行的，微观上看是交替运行的
  • 考点：单核CPU同一时刻只能执行一个程序，各个程序只能并发执行；多核CPU中，多个程序可以并行执行
  • 实例：当你在听音乐的同时浏览网页，实际上是CPU在多个任务之间快速切换，给人同时运行的感觉。

2. 共享：

  • 指系统中的资源供内存中多个并发执行的进程共同使用
  • 互斥共享方式：资源被多个进程依次使用（如打印机）
  • 同时共享方式：宏观上"同时"，微观上交替地对资源进行访问（如磁盘文件）
  • 有时微观上也确实是同时发生的（如多核CPU访问内存）
  • 实例：多个用户可以同时访问同一个网站，这就是资源共享的体现；多个程序可以同时使用硬盘存储数据。

3. 虚拟：

  • 将一个物理上的实体变为若干个逻辑上的对应物
  • 物理实体是实际存在的，而逻辑上的对应物是用户感受到的
  • 虚拟存储器技术：如电脑只有4GB内存，而GTA5需要4GB内存、QQ需要256MB内存，它们可以同时运行，这是因为虚拟存储器技术
  • 虚拟处理器技术：如单核CPU中，用户同时打开QQ、网易云、微信等软件，实际上只有一个CPU，但用户感觉有多个CPU在服务
  • 实例：虚拟机软件（如VMware、VirtualBox）可以在一台物理机器上运行多个操作系统；云计算中的虚拟服务器。

4. 异步：

  • 多道程序环境下，进程的执行不是一贯到底的，而是走走停停，以不可预知的速度向前推进
  • 每个程序什么时候执行、向前推进速度快慢，由执行的现场决定
  • 同一程序在相似的初始数据下，无论何时运行都应获得同样的结果
  • 实例：当你运行一个程序时，它的执行速度可能会因为其他程序的运行而变化，这就是异步性的体现。

1.1.3 操作系统的目标和功能

操作系统的目标

  1. 方便性：为用户提供友好的界面，使用户能更方便地使用计算机
  2. 有效性：提高系统资源利用率，提高系统吞吐量
  3. 可扩充性：便于系统功能的扩展和修改
  4. 开放性：遵循标准和规范，便于系统间的互操作

实例：Windows的图形界面提供了方便性；任务管理器展示了资源使用情况体现了有效性；插件系统体现了可扩充性；网络协议支持体现了开放性。

操作系统的基本功能

  1. 作为计算机系统资源的管理者：
    • 处理机管理：进程调度、进程控制、进程同步与通信
    • 存储器管理：内存分配回收、内存保护、地址映射、内存扩展
    • 文件管理：文件存储空间的管理、文件操作的一般管理、目录管理、文件的读写管理和存取控制
    • 设备管理：设备的分配与回收、设备驱动程序、设备无关性、缓冲区管理
  2. 作为用户与计算机硬件系统之间的接口：
    • 命令接口：用户通过命令与系统交互
    • 联机命令接口：如Windows + R打开"黑窗口"输入命令
    • 脱机命令接口：通过文件提交作业
    • 程序接口：提供系统调用（如文件操作、进程控制等系统调用）
    • 图形接口：如Windows桌面、MacOS界面
  3. 实现对计算机资源的扩充：
    • 通过虚拟技术，使用户感觉拥有比实际更多的资源
    • 例如：虚拟存储器技术使用户感觉有比实际内存更大的内存空间
1.2 操作系统的发展历程

1.2.1 手工操作阶段（无操作系统）

特点：用户直接操作计算机，没有操作系统

缺点：

  • 人机矛盾：用户需要等待计算机完成任务
  • CPU利用率低：CPU和I/O设备之间速度不匹配
  • 作业调度困难：用户需要手动操作

实例：早期的大型计算机需要操作员手动加载程序卡片，每张卡片代表一条指令，非常低效。

1.2.2 批处理阶段（操作系统开始出现）

单道批处理系统：

  • 内存中始终只存放一个作业
  • 优点：提高了CPU利用率
  • 缺点：仍存在人机矛盾，CPU利用率不高

实例：IBM的早期批处理系统，将一批作业预先准备好，然后一次性处理。

多道批处理系统：

  • 允许多个程序同时进入内存并允许它们在CPU中交替运行
  • 优点：资源利用率高，系统吞吐量大
  • 缺点：没有交互能力，平均周转时间长

实例：现代的后台批处理任务，如数据备份、报表生成等，不需要用户交互。

1.2.3 分时操作系统（UNIX）

特点：

  • 多路性：系统允许多台终端同时连接到一台主机上
  • 交互性：用户可以通过终端与系统进行广泛的人机对话
  • 及时性：用户的请求能在很短时间内被响应（通常1-3秒）
  • 独立性：每个用户在各自的终端进行操作，互不干扰

优点：解决了人机交互问题

实例：UNIX系统允许多个用户通过不同的终端同时登录并执行各自的任务。

1.2.4 实时操作系统

特点：

  • 即时响应：计算机系统接收到外部信号后及时进行处理
  • 高可靠性：必须在严格时限内完成事件处理

应用领域：军事、工业控制、航空航天等

实例：汽车的防抱死制动系统(ABS)、飞机的自动驾驶系统、导弹制导系统等都需要实时操作系统。

1.2.5 网络操作系统和分布式计算机系统

网络操作系统：

  • 把计算机网络中的各台计算机有机地结合起来
  • 提供一种统一的、经济而有效的使用各计算机的方法
  • 实现各台计算机之间数据的相互传送

实例：Windows Server、Novell NetWare等，用于构建局域网环境。

分布式操作系统：

  • 系统中任意两台主机之间使用通信方式交换信息
  • 系统中任意台主机都可以构成一个子系统
  • 每台计算机地位相等
  • 每台计算机上的资源都被所有用户共享
  • 任务可以分布在几台计算机上由它们并行、协同完成

特点：并行性和分布性

实例：Google的分布式文件系统、Apache Hadoop大数据处理平台。

1.2.6 个人计算机操作系统

特点：

  • 为个人计算机设计的操作系统
  • 通常具有友好的图形用户界面
  • 支持多任务、多用户

代表系统：Windows、MacOS、Linux

实例：我们日常使用的个人电脑、笔记本电脑上安装的Windows 11、macOS Ventura等。

1.3 操作系统运行环境

1.3.1 处理器运行模式

内核态（核心态、管态）：

  • CPU执行特权指令时的状态
  • 可以访问所有系统资源和执行所有指令
  • 操作系统在内核态运行

实例：当程序需要访问硬件设备或执行关键系统操作时，必须切换到内核态。

用户态（目态）：

  • CPU执行非特权指令时的状态
  • 只能访问有限的系统资源和执行非特权指令
  • 应用程序在用户态运行

实例：用户编写的普通应用程序（如文本编辑器、浏览器）都在用户态运行。

状态转换：

  • 内核态→用户态：执行一条特权指令，修改PSW的标志位变为"用户态"，操作系统主动让出CPU使用权
  • 用户态→内核态：触发中断信号，操作系统强行夺回CPU使用权

实例：当你在浏览器中点击打印按钮时，浏览器（用户态）会发起系统调用，CPU切换到内核态执行打印操作，完成后返回用户态继续运行浏览器。

CPU运行模式转换图

用户态 ←─────────── 特权指令 ───────────── 内核态
 │                                          │
 │               中断/异常                  │
 └──────────────────────────────────────────┘

说明：用户态通过系统调用或中断进入内核态，内核态通过特权指令返回用户态

1.3.2 中断和异常的概念

中断：

  • 外部事件引起的处理器暂停当前任务，转而执行中断处理程序

实例：当你按下键盘按键时，会产生键盘中断，CPU暂停当前任务去处理按键输入。

分类：

  • 内中断（异常）：与当前执行的指令有关（如除零错误、缺页中断）
  • 外中断（狭义上的中断）：与当前执行的指令无关（如键盘输入、定时器中断）

实例：程序试图除以零时会发生除零异常（内中断）；网卡接收到数据包时会产生网络中断（外中断）。

记忆方法：

关键词记忆法：

  • 内中断（Internal Interrupt） - "内"表示内部，与程序内部执行的指令有关，如程序错误
  • 外中断（External Interrupt） - "外"表示外部，来自CPU外部的硬件信号，如键盘、鼠标等外设

联想记忆法：

  • 内中断：程序"内心"出现问题，如计算错误、非法指令
  • 外中断：外界"干扰"，如用户按键、定时器到时

中断的作用：

  • 让用户态变回内核态的唯一方式
  • 让操作系统强行夺回CPU的控制权
  • 没有中断就没有并发

实例：系统调用通过软中断实现从用户态到内核态的切换。

中断机制的基本实现原理：

  1. 外部事件发生
  2. 硬件检测到中断信号
  3. CPU暂停当前任务，保存当前状态
  4. 转到中断处理程序
  5. 执行中断处理程序
  6. 恢复被中断任务

实例：当你移动鼠标时，鼠标会产生中断信号，CPU暂停当前工作，执行鼠标中断处理程序更新光标位置，然后继续原来的工作。

    ┌─────────────┐
    │  CPU执行程序  │
    └─────────────┘
           │
    外部事件发生    ↓
           │
    ┌─────────────┐
    │  硬件检测中断  │
    └─────────────┘
           │
      保存CPU状态    ↓
           │
    ┌─────────────┐
    │ 跳转到中断处理程序│

    ┌─────────────┐
    │ 继续执行原程序 │
    └─────────────┘

1.3.3 系统调用

定义：操作系统提供给应用程序的接口，用于请求操作系统服务

实例：C语言中的printf()函数底层会调用系统提供的write()系统调用来输出字符到屏幕。

系统调用的实现：

  1. 用户程序调用系统调用指令
  2. 触发中断，CPU切换到内核态
  3. 操作系统执行相应的系统调用服务例程
  4. 返回结果给用户程序
  5. 恢复用户程序执行

实例：当你在程序中调用fopen()打开文件时，实际是通过系统调用请求操作系统打开指定文件。

系统调用实现过程图

    ┌─────────────┐
    │  用户程序运行  │
    └─────────────┘
           │
    调用系统调用指令  ↓
           │
    ┌─────────────┐
    │   触发中断    │───→ CPU切换到内核态
    └─────────────┘
           │
    ┌─────────────┐
    │ 执行系统调用服务 │
    │    例程      │
    └─────────────┘
           │
    返回结果给用户程序 ↓
           │
    ┌─────────────┐
    │ 恢复用户程序执行 │←── CPU切换回用户态
    └─────────────┘

系统调用的分类：

  • 进程控制：创建、终止、挂起、恢复
  • 文件操作：打开、关闭、读、写
  • 设备管理：读、写、控制
  • 信息维护：获取时间、设置时间
  • 通信：消息传递、共享内存

实例：fork()用于创建进程，read()/write()用于文件读写，socket()用于网络通信。

1.4 操作系统结构

分层结构：

  • 将操作系统分为若干层，每层仅依赖于下一层
  • 优点：结构清晰，易于开发和维护
  • 缺点：各层之间通信效率低

实例：THE操作系统是典型的分层结构，分为硬件层、处理器分配层、存储器和I/O设备管理层等。

模块化结构：

  • 将操作系统分解为若干功能模块
  • 模块间通过接口通信
  • 优点：模块化开发，便于维护
  • 缺点：模块间接口设计复杂

实例：现代操作系统通常采用模块化设计，如Linux内核的不同功能模块。

宏内核：

  • 整个操作系统作为一个大内核
  • 优点：运行效率高
  • 缺点：结构复杂，不易维护

实例：传统的UNIX系统和Linux内核都属于宏内核结构。

微内核：

  • 仅将最核心的功能（进程管理、内存管理）放在内核中
  • 其他功能（文件系统、设备驱动）作为用户级服务
  • 优点：结构清晰，易于扩展
  • 缺点：通信开销大，效率相对较低

实例：Windows NT、macOS XNU内核、QNX等采用了微内核或混合内核设计。

外核：

  • 将操作系统分为内核和外核
  • 外核负责资源管理，内核负责执行
  • 优点：结构灵活，安全性高
  • 缺点：实现复杂

实例：MIT的Exokernel是外核结构的典型代表。

1.5 操作系统引导

引导过程：

  1. CPU上电后，执行ROM中的BIOS引导程序
  2. BIOS引导程序指示CPU将磁盘中的MBR（主引导记录）读入内存
  3. CPU执行磁盘引导程序
  4. 磁盘引导程序根据分区表判断C盘位置
  5. 读入C盘中的PBR（分区引导记录）程序并执行
  6. CPU找到C盘根目录中的启动管理器程序，读入主存并执行
  7. 启动管理器完成操作系统初始化工作

实例：当你按下电脑电源按钮时，计算机会依次执行上述步骤来启动Windows或Linux系统。

           │
    ┌─────────────┐

    └─────────────┘

关键概念：

  • MBR：磁盘开头的位置留出的区域，用于存储引导程序
  • 分区表：数据结构，说明磁盘中每个分区的大小和地址范围
  • ROM：只读存储器，断电数据不丢失，存储BIOS
  • RAM：随机存取存储器，断电数据丢失，存储操作系统

实例：使用DiskGenius等工具可以查看和编辑磁盘的MBR和分区表信息。

1.6 虚拟机

1.6.1 虚拟机的基本概念

虚拟机：通过软件模拟的计算机系统，具有完整的硬件功能

实例：VMware Workstation、Oracle VirtualBox、Microsoft Hyper-V等软件可以创建和运行虚拟机。

虚拟机管理程序（VMM）：

  • 第一类VMM：直接运行在硬件上，没有宿主操作系统
    • 优点：性能高
    • 缺点：管理复杂
  • 第二类VMM：运行在宿主操作系统之上
    • 优点：管理简单
    • 缺点：性能较低

实例：VMware ESXi是第一类VMM；VMware Workstation是第二类VMM。

虚拟化原理：

  • 通过VMM将物理资源（CPU、内存、I/O设备）抽象为虚拟资源
  • 每个虚拟机看到的是虚拟的硬件
  • VMM负责将虚拟机的指令转换为物理硬件的指令

实例：在一台物理服务器上运行多个虚拟机，每个虚拟机都认为自己独占整台服务器。

虚拟机的优势：

  • 资源利用率高：一台物理机可以运行多个虚拟机
  • 安全性高：虚拟机之间相互隔离
  • 管理便捷：可以快速创建、删除、迁移虚拟机

实例：云计算服务商使用虚拟机技术为用户提供弹性计算服务，如AWS EC2、阿里云ECS。

1.7 本章疑难点

并发与并行的区别：

  • 并发：宏观上同时发生，微观上交替发生
  • 并行：同一时刻同时发生

实例：单核CPU上的多任务是并发执行；多核CPU上同时运行多个任务是并行执行。

共享的两种方式：

  • 互斥共享：资源被多个进程依次使用
  • 同时共享：宏观上"同时"，微观上交替地对资源进行访问

实例：打印机是互斥共享资源；内存是同时共享资源。

虚拟与实际的区别：

  • 虚拟：用户感受到的逻辑上的资源
  • 实际：物理存在的资源

实例：虚拟内存是用户感受到的大于实际物理内存的存储空间。

中断与异常的区别：

  • 内中断（异常）：与当前执行的指令有关
  • 外中断：与当前执行的指令无关

实例：程序执行除零操作引发异常；键盘按键产生外部中断。

操作系统引导过程：

从ROM引导程序到BIOS，再到MBR，再到PBR，最后到启动管理器

实例：UEFI启动过程替代了传统的BIOS+MBR方式，提供了更快更安全的启动体验。

操作系统结构：

分层结构、模块化结构、宏内核、微内核、外核的优缺点比较

实例：Linux采用宏内核但模块化设计；Windows NT采用混合内核设计。
